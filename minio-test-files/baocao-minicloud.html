1. Giới thiệu tổng quan

MyMiniCloud là một dự án xây dựng hệ thống Cloud Computing thu nhỏ, được phát triển hoàn toàn bằng công nghệ Docker và Docker Compose. Hệ thống mô phỏng đầy đủ 9 thành phần cốt lõi của một nền tảng đám mây thực tế như Amazon Web Services (AWS), Microsoft Azure, hoặc Google Cloud Platform (GCP).

Mục tiêu dự án

  • Hiểu rõ kiến trúc và cách hoạt động của Cloud Platform
  • Thực hành triển khai hệ thống phân tán với Docker
  • Áp dụng các công nghệ DevOps và Container Orchestration
  • Xây dựng kiến thức về Microservices Architecture

2. Kiến trúc hệ thống

Hệ thống MyMiniCloud bao gồm 9 server chính, mỗi server đảm nhận một vai trò cụ thể và giao tiếp với nhau qua Docker Network nội bộ (cloud-net):

STT Tên Server Công nghệ Cổng Chức năng
1 Web Frontend Nginx (stable) 8080 Phục vụ website tĩnh, blog cá nhân
2 App Backend Flask (Python 3.11) 8085 REST API server, xử lý business logic
3 Database MariaDB 11 3306 Lưu trữ dữ liệu quan hệ (RDBMS)
4 Authentication Keycloak (latest) 8081 Identity Provider, OIDC/OAuth2.0
5 Object Storage MinIO (latest) 9000/9001 S3-compatible object storage
6 Internal DNS Bind9 (v9.18) 9053 DNS resolution cho services nội bộ
7 Monitoring Prometheus 9090 Thu thập và lưu trữ metrics
8 Dashboard Grafana 3000 Visualization & monitoring UI
9 API Gateway Nginx (stable) 80 Reverse proxy, load balancing

3. Các tính năng đã triển khai

3.1. Tính năng cơ bản (9/9 servers)

  • Web Server: Nginx phục vụ trang chủ và routing
  • Application Server: Flask API với endpoints /hello, /secure
  • Database: MariaDB với database minicloud, bảng notes
  • Authentication: Keycloak quản lý users và tokens
  • Storage: MinIO object storage service
  • DNS: Bind9 phân giải domain nội bộ cloud.local
  • Prometheus: Monitoring với Node Exporter
  • Grafana: Dashboard visualization
  • Reverse Proxy: Nginx routing và load balancing

3.2. Các tính năng mở rộng (10/10)

1. Blog cá nhân: 3 bài viết về Cloud Computing, Docker, Microservices

2. Students API: CRUD operations cho quản lý sinh viên

3. StudentDB: Database với bảng students, 5 records mẫu

4. Keycloak Realm: myminicloud-realm với 2 users (sv01, sv02)

5. MinIO Buckets: profile-pics và documents với files uploaded

6. DNS Records: Bản ghi cho tất cả services nội bộ

7. Prometheus Targets: Monitoring web-frontend-server

8. Grafana Dashboard: System Health với 3 panels (CPU, Memory, Network)

9. Proxy Routes: /student/ endpoint routing

10. Load Balancing: Round-robin giữa 2 web servers
